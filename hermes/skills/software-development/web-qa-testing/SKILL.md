# Web QA Testing

Exploratory testing workflow for finding bugs in web applications.

## When to Use

- Testing new features before release
- Finding edge cases and bugs
- Generating bug reports with evidence

## Workflow

1. **Scope**: Define what to test (feature, page, flow)
2. **Explore**: Navigate, click, input, break things
3. **Document**: Capture screenshots, steps to reproduce
4. **Report**: Structured bug report with severity

## Tools

- Browser navigation and interaction
- Screenshots for evidence
- Console errors and network failures

## Output

Bug reports with:
- Reproduction steps
- Expected vs actual behavior
- Severity assessment
- Screenshots/logs
